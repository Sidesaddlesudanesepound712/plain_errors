# Contributing to PlainErrors

Thanks for considering contributing!

## Development Setup

```bash
git clone https://github.com/panozzaj/plain_errors.git
cd plain_errors
bundle install
bundle exec rspec
```

## Making Changes

1. Create a branch: `git checkout -b my-feature`
2. Write code and tests
3. Ensure tests pass: `bundle exec rspec`
4. Test locally in a Rails app (see `examples/README.md`)

## Pull Requests

- Open an issue first for significant changes to discuss the approach
- Write tests for new features/fixes
- Keep changes focused (one feature per PR)
- Update docs if adding configuration options
- Ensure all tests pass

## Reporting Issues

Include:
- Ruby and Rails versions
- Steps to reproduce
- Expected vs actual behavior
- Error messages

## Questions?

Open an issue for questions or discussions.
